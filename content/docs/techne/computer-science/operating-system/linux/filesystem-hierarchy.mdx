Linux 最优秀的就是他的多人任务了，那么文件权限就很重要了，一般将文件的可存取身份分为： owner、group、ohters，且都有 read、write、execute 等权限。

## 用户身份与群组记录的文件

在 linux 系统当中，默认情况下，相关信息是存储在下面的目录下的

- `/etc/passwd`：账户信息（目前笔者还不知道里面有具体信息，看不太懂）

- `/etc/shadow`：账户密码，不过是加密了的

- `/etc/group`：所有组名

这三个目录可以说是 Linux 系统里面账户、密码、群组信息的集中地，不要随便删除

| 目录         | 功能简述                                                                 |
| ------------ | ------------------------------------------------------------------------ |
| /bin         | 存放常用的二进制可执行文件（基本命令：ls, cp, mv, cat…），普通用户可用。 |
| /sbin        | 系统管理员用的二进制工具（ifconfig, shutdown…）。                        |
| /boot        | 引导加载程序相关文件（如 vmlinuz 内核、grub 引导）。                     |
| /dev         | 设备文件（硬盘 /dev/sda，终端 /dev/tty，随机数 /dev/random）。           |
| /etc         | 系统配置文件（passwd, hosts, fstab 等）。                                |
| /home        | 普通用户的家目录（如 /home/alice）。                                     |
| /lib, /lib64 | 系统运行所需的共享库文件（类似 Windows DLL）。                           |
| /media       | 移动设备挂载点（U盘、光盘自动挂载）。                                    |
| /mnt         | 临时挂载点，管理员手动挂载磁盘/分区用。                                  |
| /opt         | 第三方软件安装位置（比如大软件包）。                                     |
| /proc        | 伪文件系统，内存中的内核和进程信息（如 /proc/cpuinfo）。                 |
| /root        | root 用户的主目录。                                                      |
| /run         | 系统运行时产生的临时文件（重启会清空）。                                 |
| /srv         | 服务相关数据（如 web 服务的页面数据）。                                  |
| /sys         | 内核对象接口，现代内核信息访问点（设备、驱动）。                         |
| /tmp         | 临时文件存放，重启清空。                                                 |
| /usr         | 用户程序与库的主要存放处（/usr/bin, /usr/lib, /usr/share）。             |
| /var         | 可变数据（日志 /var/log、缓存 /var/cache、邮件、队列）。                 |
