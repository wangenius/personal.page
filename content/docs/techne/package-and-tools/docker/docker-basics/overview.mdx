---
title: Docker 基础与日常工作流
---

这一模块的目标是：在已有本地环境（如 Colima）之上，快速掌握 Docker 最常用的一套日常工作流，做到：

- 能够读懂与编写基础 `Dockerfile`。
- 理解镜像与容器的关系，以及如何用 `docker run` / `docker exec` 调试。
- 使用 Docker Compose 组织多服务本地联调。
- 掌握将镜像推送到仓库、从仓库分发到 CI/服务器的基本流程。

## 前置条件

- 已在本地安装并启动 Docker 环境（推荐基于 Colima，或其他兼容 Docker CLI 的方案）。
- 了解基础命令行操作（`cd`、`ls`、`bash` 等）。

如果你尚未完成本地环境搭建，建议先阅读《Colima 本地容器环境》模块，再回到本章节。

## 本模块涵盖内容

1. 镜像与容器：
   - 镜像是什么、容器是什么，两者如何对应。
   - 常用命令：`docker images`、`docker ps`、`docker run`、`docker exec` 等。
   - 通过一个简单示例体验完整生命周期。

2. 构建镜像与缓存策略：
   - 从零写一个 `Dockerfile`，引入多阶段构建与 `.dockerignore`。
   - 理解构建缓存如何工作，避免每次都“从头来过”。

3. Docker Compose 联调：
   - 用一份 `docker-compose.yml` 管理多个服务。
   - 支持代码热更新与不同环境（开发 / CI）之间的配置切换。

4. 镜像仓库与分发：
   - 登录、打标签、推送到远程仓库。
   - 多架构镜像构建与推送。
   - 版本标签与回滚策略的基本原则。

## 在整体路线中的位置

- 前一站：通过 Colima 搭好本地 Docker 环境，解决“能跑”的问题。
- 本模块：学习如何高效地使用 Docker，与日常开发工作流结合。
- 下一站：进入《Docker 原理与优化》与《Kubernetes 与云端实践》，从“会用”到“理解内部机制”与“会在生产环境落地”。

读完本模块，你应该可以独立完成：

- 为一个应用写出可用且可维护的 `Dockerfile`。
- 在本地用 Docker Compose 一键拉起整个开发栈。
- 将镜像推送到指定的仓库，并在其他环境中顺利拉取与运行。
