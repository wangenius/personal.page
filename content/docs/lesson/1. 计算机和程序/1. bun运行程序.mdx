# 实际练习

目标：用 **Bun** 从 0 到 1 跑通「写一个程序并运行」的完整流程。

------

## 步骤 1：进入终端 & 新建项目目录

```bash
mkdir demo-bun
cd demo-bun
```

👉 学员会意识到：我们在「文件系统」里为项目建了一个「家」。

------

## 步骤 2：初始化项目

```
npm init
```

一路按回车，会生成：

- `package.json`
- `index.ts`（Bun 默认帮你生成一个入口文件）

------

## 步骤 3：查看入口文件

```
cat index.ts
```

你会看到类似：

```
console.log("Hello via Bun!");
```

👉 这就是程序的“起点”。

------

## 步骤 4：运行程序

```
bun run index.ts
```

输出：

```
Hello via Bun!
```

👉 这是最小可运行的程序，从 **源代码 → 文件 → 执行**，学员在 1 分钟内就能亲手完成。

------

## 步骤 5：加一个交互

编辑 `index.ts`：

```
const name = prompt("请输入你的名字: ");
console.log(`你好, ${name}!`);
```

再次运行：

```
bun run index.ts
```

👉 现在程序能和人类互动了。

------

## 步骤 6：尝试一个模块（感受生态）

Bun 支持直接安装依赖：

```bash
npm install dayjs
```

修改 `index.ts`：

```js
import dayjs from "dayjs";

console.log("当前时间是:", dayjs().format("YYYY-MM-DD HH:mm:ss"));
```

运行：

```
node index.js
```

👉 学员会看到当前时间。
 这一步的目标是：**程序不仅能跑，还能调用“外部能力”**（模块/库），真正具备生产力。
