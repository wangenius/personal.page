# 解构项目（Deconstruct Projects）

**目标：通过拆解不同类型的项目，理解一个软件项目是如何从源代码到实际运行的。**

## 一、环境：程序运行的土壤

任何项目都离不开两个环境：

1. **运行环境（Runtime / 编译器）**
   - 用来把人类语言（代码）翻译成机器能执行的指令。
   - 例子：
     - JS → Node.js / Bun / Deno
     - Python → Python 解释器
     - Rust → Rustc
2. **包管理器（Package Manager）**
   - 项目不可能从零开始，总要依赖第三方库。
   - 包管理器的作用是帮你快速下载和管理这些依赖。
   - 例子：
     - Node.js → npm / yarn / pnpm
     - Python → pip
     - Rust → cargo

👉 小结：**环境决定了代码能不能跑，包管理器决定了项目能不能做大**。

## 二、从一个 Bun 脚本开始

最小的 Bun 程序：

```ts
// index.ts
console.log("hello, world");
```

运行：

```bash
bun run index.ts
```

👉 你已经完成了一个 **CLI 程序**。
 流程：**源代码 → Bun 翻译 → CPU 执行**。

## 三、一个 Bun 项目

新建项目：

```bash
mkdir demo-bun
cd demo-bun
bun init
```

👉 `bun init` 会生成：

- `package.json`
- `tsconfig.json`
- `index.ts`

安装第三方依赖：

```bash
bun add axios
```

使用它：

```ts
import axios from "axios";

const res = await axios.get("https://api.github.com");
console.log(res.data);
```

运行：

```bash
bun run index.ts
```

👉 小结：**有了包管理器，项目就能调用外部能力**。

## 四、一个前端项目

前端的本质：浏览器已经是一个渲染程序，能执行 HTML / CSS / JS。
 我们只需要写能被浏览器理解的代码。

现代前端开发通常使用 **React + Vite**。

创建 React 项目（使用 Vite 模板）：

```bash
bun create vite frontend
cd frontend
bun install
bun run dev
```

👉 打开浏览器访问 `http://localhost:5173`，就能看到运行的前端页面。

> HTML：页面结构
>  CSS：页面样式
>  JS：交互逻辑
>  React/Vite：现代前端框架 & 构建工具

## 五、一个后端项目（Bun 服务器）

Bun 内置了 HTTP 服务器：

```ts
// server.ts
Bun.serve({
  port: 3000,
  fetch(req) {
    return new Response("Hello from Bun backend!");
  },
});
```

运行：

```bash
bun run server.ts
```

访问 `http://localhost:3000`，就能看到返回结果。

👉 这是一个最小的后端 API 服务。

------

## 六、前后端分离

- **前端**：用 Vite + React 提供用户界面
- **后端**：用 Bun.serve 提供 API
- **前端调用后端 API**：获取或提交数据

前端调用后端示例：

```ts
// 在 React 组件里
const res = await fetch("http://localhost:3000");
const data = await res.text();
console.log(data); // "Hello from Bun backend!"
```

👉 小结：**前端 = 展示层，后端 = 服务层**。
