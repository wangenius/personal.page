## 一、后端是什么？

1. **前端 vs 后端**
   - 前端：运行在浏览器里，负责展示 UI。
   - 后端：运行在服务器上，负责处理逻辑、存储数据、提供 API。
2. **为什么需要后端？**
   - 负责“数据”和“安全”。
   - 用户提交表单 → 后端存到数据库 → 前端再展示出来。

👉 小结：**前端是“脸”，后端是“大脑”。**

------

## 二、6.1 用 Node.js Express 创建一个后端

### 安装依赖

```bash
mkdir backend-demo
cd backend-demo
npm init -y
npm install express
```

### 写一个最小的服务

```
index.js
const express = require("express");
const app = express();

app.get("/", (req, res) => {
  res.send("Hello from Express!");
});

app.get("/time", (req, res) => {
  res.json({ time: new Date().toISOString() });
});

app.listen(3000, () => {
  console.log("服务已启动：http://localhost:3000");
});
```

运行：

```bash
node index.js
```

✅ 打开浏览器访问：

- `http://localhost:3000/` → 显示 `"Hello from Express!"`
- `http://localhost:3000/time` → 返回 JSON

👉 学员就完成了一个 **最小 API 服务**。

## 三、6.2 包 (Package) 管理

- **为什么需要包？**
  - 不可能所有功能都自己写。
  - 使用 npm（Node 的包管理器）快速安装第三方库。
- **常见例子**：
  - `express`：服务器框架
  - `axios`：HTTP 请求
  - `dotenv`：环境变量
  - `cors`：跨域支持

👉 小结：**包管理器让后端开发像“搭积木”。**

------

## 四、6.3 部署你的后端

目标：把服务从本地跑到互联网上。

### 方式 1：用 Vercel 部署（最简单）

1. 推送代码到 GitHub。
2. 在 [Vercel](https://vercel.com/) 连接仓库。
3. Vercel 自动识别 Node 项目 → 一键部署。

👉 访问你的域名：`https://your-app.vercel.app`

------

### 方式 2：用 Render / Railway（更接近真实服务器）

1. 注册 [Render](https://render.com/) 或 [Railway](https://railway.app/)。
2. 新建 Web Service → 连接 GitHub 仓库。
3. 设置启动命令：

```bash
node index.js
```

1. 部署完成后，平台会分配一个公网地址。

👉 小结：**部署平台相当于“帮你租了一台云服务器”。**

------

## 五、6.4 网络环境与域名

- **IP 地址**：服务器的“门牌号”。
- **端口 (Port)**：服务的“房间号”。
- **域名 (Domain)**：给地址起一个好记的名字。

👉 例子：

- `http://127.0.0.1:3000` = 本地开发环境
- `https://myapp.vercel.app` = 部署后的公网地址
- 自定义域名：`https://myapp.com`
